# حزمة التحويل التنفيذية — منصة مساعد مدير التصوير

هذه الحزمة تحوّل التصور النظري إلى ملفات تشغيلية أولية قابلة للإسقاط داخل مستودع هندسي.

## المحتويات

- `contracts/openapi/openapi.yaml`
  - مواصفة OpenAPI 3.1 التأسيسية للواجهة الخلفية.
- `db/migrations/*.sql`
  - مخطط قاعدة البيانات، التوابع، الفهارس، سياسات العزل، والـseeds.
- `agents/definitions/*.json`
  - تعريفات الوكلاء وسياسات القراءة والكتابة والمراجعة.
- `docs/*.md`
  - وثائق مساندة: PRD مختصر، نموذج البيانات، Runtime الوكلاء، والـbacklog.

## ملاحظات تنفيذية

1. هذه الحزمة تمثّل baseline هندسياً قوياً، وليست المنتج النهائي.
2. المواصفة تركّز على اتساق العقود، النسخ، الاعتماد، والعزل المؤسسي.
3. ملفات الـmigration مكتوبة على PostgreSQL مع `pgvector` و`pg_trgm`.
4. الوكلاء لا يكتبون مباشرةً فوق الكيانات المعتمدة، بل عبر مسودات أو كيانات قابلة للمراجعة.
5. بعض أجزاء التكامل مع مزودي الكاميرا/الـDIT مقصود أن تبقى ضمن `metadata_json` وadapters مستقلة.

## الترتيب الموصى به للتنفيذ

1. تشغيل `db/migrations`.
2. نشر `contracts/openapi/openapi.yaml` في بوابة العقود.
3. seed للوكلاء من `db/migrations/0015_seed_agents.sql`.
4. بناء workers وفق `agents/definitions`.
5. ربط الـAPI بالـRLS عبر تمرير `app.org_id` داخل جلسة قاعدة البيانات.

## الخطوة التالية

بعد إدخال هذه الحزمة إلى المستودع، المنطقي هو:

- توليد SDK من ملف OpenAPI.
- بناء طبقة repository/service لكل domain.
- تنفيذ اختبارات contract وmigration وpolicy enforcement.
