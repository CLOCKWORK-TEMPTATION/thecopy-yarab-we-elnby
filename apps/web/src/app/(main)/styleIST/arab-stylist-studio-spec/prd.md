# PRD - Arab Stylist Studio

**التاريخ المرجعي:** 2026-03-28  
**نوع الوثيقة:** Product Requirements Document  
**حالة الوثيقة:** Draft for implementation

---

## 1. تعريف المنتج

Arab Stylist Studio هو تطبيق ويب تفاعلي مخصص للأفلام والمسلسلات العربية. الهدف منه أن يعمل كمنصة تشغيل فعلية لفريق الستايلست والملابس والهوية البصرية للشخصيات، لا كمجرد واجهة دردشة.

يقوم المنتج على أربع طبقات رئيسية:

1. تحليل السيناريو واستخراج الشخصيات والمشاهد والإشارات البصرية.
2. بناء **Style Bible** لكل شخصية.
3. اقتراح **Looks** على مستوى المشهد وربطها بالمراجع والميزانية.
4. إدارة الاستمرارية، التعليقات، الموافقات، والنسخ المعتمدة.

---

## 2. المشكلة

في بيئات الإنتاج العربية، تنتشر هذه المشكلات:

- العمل موزع بين ملفات PDF وصور ومحادثات ورسائل غير قابلة للتتبع.
- لا توجد مرجعية موحدة للهوية البصرية للشخصيات.
- الاستمرارية البصرية بين المشاهد والحلقات عرضة للكسر.
- لا يوجد سجل واضح للموافقة أو الرفض أو سبب التعديل.
- الوصول السريع إلى المراجع الملائمة يتم يدويًا وببطء.

---

## 3. الرؤية

تحويل السيناريو العربي والمرجعيات البصرية وقرارات الفريق إلى **نظام عمل موحّد** يمكنه:

- فهم النص،
- اقتراح الهوية البصرية،
- دعم القرار البشري،
- حفظ أثر تدقيقي كامل لكل تعديل وقرار.

---

## 4. الفرضية التشغيلية

إذا تم بناء المنتج على **نموذج بيانات صريح** وربط الذكاء الاصطناعي بهذا النموذج عبر وكلاء محددين وأدوات محددة، فستصبح مخرجاته قابلة للتنفيذ والمراجعة والتتبع، بدل أن تبقى مجرد ردود نصية متفرقة.

## 8. حالات الاستخدام الجوهرية

### U1 - إنشاء مشروع

كمستخدم إداري، أريد إنشاء مشروع فيلم أو مسلسل، حتى تُنظم الحلقات والمشاهد والملفات تحت نطاق واحد.

### U2 - استيراد سيناريو

كمستخدم إبداعي، أريد رفع نص الحلقة أو الفيلم، حتى أبدأ استخراج المشاهد والشخصيات والإشارات البصرية.

### U3 - إنشاء ملف شخصية

كمستخدم Stylist، أريد أن يبني النظام ملفًا بصريًا أوليًا للشخصية، حتى أراجعه وأعدله.

### U4 - اقتراح Looks للمشهد

كمستخدم Stylist، أريد اقتراح عدة لوكات لمشهد محدد، حتى أختار الأنسب أو أطلب تعديلًا.

### U5 - بناء Moodboard

كمستخدم إبداعي، أريد تجميع المراجع والصور المقترحة في لوحة منظمة، حتى يمكن مناقشتها واعتمادها.

### U6 - مراجعة الاستمرارية

كمستخدم Script Supervisor أو Stylist، أريد مقارنة المشاهد المتجاورة سرديًا وزمنيًا لاكتشاف أي تضارب.

### U7 - إدارة الموافقات

كمخرج أو منتج، أريد تسجيل قرار بالموافقة أو الرفض أو طلب التعديل مع تعليق محفوظ.

### U8 - البحث العربي

كمستخدم، أريد أن أبحث بالعربية عن لون أو خامة أو شخصية أو مشهد أو إكسسوار، حتى أصل بسرعة لما أحتاجه.

- مواسم متعددة
- تعدد أعضاء الفريق

كل تغيير أو قرار يجب أن يحمل:

- من
- متى
- على ماذا
- الحالة السابقة
- الحالة اللاحقة
- هل التغيير من AI أم من إنسان

## 11. المبادئ الحاكمة للذكاء الاصطناعي

1. الذكاء يقترح ولا يعتمد نهائيًا.
2. القرار النهائي البشري إلزامي في:
   - Style Bible
   - Look نهائي
   - Moodboard نهائي
   - حل مشكلة استمرارية
3. كل وكيل يعيد JSON مضبوطًا.
4. كل وكيل ممنوع من الكتابة المباشرة إلى قاعدة البيانات.
5. كل كتابة تمر عبر service/tool layer.
6. يجب حفظ كل AI run.
7. يجب إمكان تقييم كل وكيل مستقلًا.

## 19. معايير القبول للنسخة الأولى

لكي تُعتبر النسخة الأولى مكتملة، يجب أن يتحقق الآتي:

1. إنشاء مشروع جديد وتشغيله دون تدخل يدوي في قاعدة البيانات.
2. رفع سيناريو وتحويله إلى Scenes وCharacters.
3. إنشاء Style Bible واحدة على الأقل لكل شخصية رئيسية.
4. إنشاء Looks لمشهد محدد وربطها بالشخصيات.
5. بناء Moodboard وتخزينه.
6. إضافة تعليقات وتسجيل قرار اعتماد أو رفض.
7. تشغيل فحص استمرارية على مستوى الحلقة.
8. البحث بالعربية داخل النصوص والمراجع.
9. تسجيل جميع AI runs وربطها بالكيانات الصحيحة.
10. فرض صلاحيات tenant isolation.

## 20. قرارات تصميم حاسمة

- المنتج ليس chatbot.
- الـ API ليس مجرد passthrough لمزود الذكاء.
- الوكلاء ليسوا مخولين بالاعتماد النهائي.
- البحث العربي جزء أساسي من المنتج وليس إضافة لاحقة.
- كل شيء يجب أن يكون versioned.
- كل شيء يجب أن يكون auditable.
