## 1) النطاق

- الاختبار محصور في تطبيق brain-storm-ai فقط على الرابط أعلاه.
- ممنوع توسيع الاختبار إلى أي صفحة أخرى داخل المنصة.
- ممنوع الخلط مع أي تطبيق وكلاء آخر مثل development أو analysis.
- أي انتقال إلى تسجيل الدخول أو رفع ملف أو تصدير يُسمح به فقط إذا انطلق من داخل التطبيق وكان مرتبطًا بوظيفة من وظائفه المباشرة.
- أي خلل خارج التطبيق لا يُسجل إلا إذا منع اختبار التطبيق نفسه بشكل مباشر.
- ممنوع تعديل كود التطبيق أو إصلاح أي عطل.

---

## 2) قواعد حاكمة غير قابلة للتفاوض

- لا تعتمد على قراءة الكود كدليل نجاح أو فشل، الدليل الوحيد المقبول هو ما يُلتقط فعليًا من المتصفح.
- لا تعتبر مجرد وجود زر أو عنصر دليل نجاح؛ يجب التفاعل معه والتحقق من نتيجة فعلية.
- لا تعتبر تحميل الصفحة وحده نجاحًا.
- لا تستخدم تقارير سابقة كدليل نجاح أو فشل.
- لا تُسقط حالة اختبار لأن الميزة غير واضحة؛ ابحث عن النظير الوظيفي.
- وثّق كل فشل بدليل مرئي أو سجل Console أو Network أو HAR.
- استخدم حالات: ناجح، فاشل، محجوب، غير موجود فقط.
- ممنوع استخدام "ناجح جزئيًا".
- ممنوع تشغيل Playwright أو Cypress أو أي إطار اختبار آلي.
- ممنوع استخدام Docker أو اقتراحه.

## 3) إعداد التنفيذ

- متصفح سطح مكتب حديث.
- دقة 1440 × 900، ثم اختبار بصري عند 768 × 1024.
- DevTools مفتوح: Console، Network، Storage، Cookies، Performance، Downloads.
- نافذة نظيفة لكل اختبار جلسات.

- استخدام Axe DevTools أو Lighthouse للوصولية.
- التقاط لقطات شاشة قبل وبعد كل خطوة حاسمة.
- تسجيل HAR لكل فشل.

## 5) مصفوفة الاختبارات العامة

### GEN-01 — التحميل الأولي
- الخطوات: افتح الرابط في نافذة نظيفة.
- المتوقع: تحميل شبكة الوكلاء وواجهة الجلسة دون أخطاء حرجة.
- الدليل المطلوب: لقطة + Console + Network.

### GEN-02 — قابلية التفاعل الأساسية
- الخطوات: تأكد من قابلية إدخال موضوع جلسة وبدئها.
- المتوقع: استجابة مباشرة لكل عنصر تفاعلي.
- الدليل المطلوب: لقطات + Console.

### GEN-03 — فحص Console
- الخطوات: ارصد Console من التحميل وحتى نهاية الجلسة.
- المتوقع: غياب أخطاء حمراء حرجة وعدم كشف مفاتيح أو رسائل داخلية.
- الدليل المطلوب: نسخ كامل لمحتوى Console.

### GEN-04 — فحص Network
- الخطوات: راقب طلبات الوكلاء والتدفقات الحية.
- المتوقع: غياب 4xx/5xx غير مبررة، عدم تسريب توكنات.
- الدليل المطلوب: HAR.

### GEN-05 — الوصولية
- الخطوات: شغّل Axe أو Lighthouse على واجهة الجلسة.
- المتوقع: غياب مشاكل Critical أو Serious.
- الدليل المطلوب: تقرير الأداة.

### GEN-06 — التنقل بلوحة المفاتيح
- الخطوات: استخدم Tab و Enter بين عناصر الجلسة والوكلاء.
- المتوقع: ترتيب منطقي للتركيز.
- الدليل المطلوب: لقطات.

### GEN-07 — الاستجابة للشاشات الأصغر
- الخطوات: اضبط الدقة على 768 × 1024.
- المتوقع: لا انكسار للتخطيط ولا اختفاء وظائف رئيسية.
- الدليل المطلوب: لقطات.

### GEN-08 — الحفظ أو الاستمرارية
- الخطوات: ابدأ جلسة، أضف أفكارًا، أعد التحميل.
- المتوقع: استرجاع الجلسة أو رسالة واضحة.
- الدليل المطلوب: لقطات + Storage.

### GEN-09 — LocalStorage و IndexedDB
- الخطوات: افحص ما يُكتب أثناء الجلسة.
- المتوقع: عدم تخزين بيانات حساسة بصيغة خام.
- الدليل المطلوب: لقطات Storage.

### GEN-10 — Cookies
- الخطوات: راقب الكوكيز قبل وبعد الجلسة.
- المتوقع: HttpOnly و Secure و SameSite حيث يلزم.
- الدليل المطلوب: لقطات Cookies.

### GEN-11 — XSS في الحقول
- الخطوات: أدخل في حقول الموضوع والأفكار:
  - `<script>alert(1)</script>`
  - `<img src=x onerror=alert(1)>`
  - `"><svg onload=alert(1)>`
- المتوقع: عرض حرفي أو رفض دون تنفيذ.
- الدليل المطلوب: لقطة + Console.

### GEN-12 — إدخال طويل جدًا
- الخطوات: أدخل وصف موضوع بطول 50,000 حرف.
- المتوقع: التطبيق لا ينهار.
- الدليل المطلوب: لقطة + Console.

### GEN-13 — انقطاع الشبكة
- الخطوات: شغّل Offline أثناء انتظار رد وكيل.
- المتوقع: رسالة واضحة دون فقدان الأفكار المحلية، وإمكانية استئناف بعد عودة الشبكة.
- الدليل المطلوب: لقطة + Network.

### GEN-14 — التصدير إن كان مدعومًا
- الخطوات: صدّر نتائج جلسة بكل الصيغ المتاحة.
- المتوقع: ملفات صالحة بأحجام معقولة.
- الدليل المطلوب: ملفات + لقطات.

### GEN-15 — الاستيراد أو رفع الملفات إن كان مدعومًا
- الخطوات: ارفع ملفًا صالحًا، ملفًا غير مدعوم، ملفًا فارغًا، ملفًا ضخمًا.
- المتوقع: قبول الصالح ورفض البقية برسائل واضحة.
- الدليل المطلوب: لقطات + Network.

### GEN-16 — المصادقة
- الخطوات: ادخل بحساب صالح، اختبر الخروج، حاول الوصول مجددًا.
- المتوقع: انتقالات سليمة دون تسرب جلسة.
- الدليل المطلوب: لقطات + Storage + Cookies.

### GEN-17 — الروابط والمشاركة
- الخطوات: انسخ روابط جلسات داخلية إن وجدت.
- المتوقع: عمل الروابط في نافذة جديدة.
- الدليل المطلوب: لقطات.

### GEN-18 — الأداء
- الخطوات: قِس زمن استجابة الوكلاء.
- المتوقع: عدم وجود حواجز أداء حرجة.
- الدليل المطلوب: تقرير Performance.

### GEN-19 — حالات الخطأ
- الخطوات: ابدأ جلسة بموضوع فارغ، اقطع الشبكة في منتصف الرد.
- المتوقع: رسائل خطأ مفهومة دون stack trace.
- الدليل المطلوب: لقطات.

### GEN-20 — عدم تسرب البيانات بين الجلسات
- الخطوات: ابدأ جلسة A، اخرج، افتح Profile جديد.
- المتوقع: غياب بيانات الجلسة السابقة.
- الدليل المطلوب: لقطات.

## 6) مصفوفة الاختبارات الخاصة بـ brain-storm-ai

### BST-01 — تحميل منصة العصف الذهني
- الخطوات: افتح الرابط وانتظر تحميل شبكة الوكلاء وواجهة الجلسة.
- المتوقع: ظهور الوكلاء وعناصر التحكم دون أخطاء.
- الدليل المطلوب: لقطة + Console.

### BST-02 — إدخال موضوع جلسة
- الخطوات: أدخل موضوعًا متوسط الطول، اضغط بدء.
- المتوقع: قبول الموضوع وانتقال الواجهة لحالة الجلسة.
- الدليل المطلوب: لقطات + Network.

### BST-03 — بدء جلسة عصف ذهني
- الخطوات: ابدأ جلسة، انتظر استجابة الوكلاء.
- المتوقع: بدء الأطوار وردود من الوكلاء خلال زمن معقول.
- الدليل المطلوب: لقطات + Network.

### BST-04 — اختبار أطوار الجلسة
- الخطوات: انتقل بين أطوار الجلسة المختلفة.
- المتوقع: تغير الواجهة والوظائف وفق الطور بشكل صحيح.
- الدليل المطلوب: لقطات لكل طور.

### BST-05 — تفاعل الوكلاء مع المدخلات
- الخطوات: أدخل توجيهات إضافية وراقب ردود الوكلاء.
- المتوقع: ردود تتفاعل مع المدخل الجديد.
- الدليل المطلوب: لقطات + Network.

### BST-06 — إضافة فكرة يدويًا
- الخطوات: أضف فكرة عبر الواجهة.
- المتوقع: ظهور الفكرة في القائمة فورًا.
- الدليل المطلوب: لقطة قبل وبعد.

### BST-07 — تعديل فكرة
- الخطوات: عدّل فكرة موجودة، احفظ.
- المتوقع: حفظ التعديل دون فقدان البيانات.
- الدليل المطلوب: لقطات + Storage.

### BST-08 — حذف فكرة إن كان مدعومًا
- الخطوات: احذف فكرة، تأكد من اختفائها.
- المتوقع: اختفاء الفكرة مع تأكيد، أو عدم وجود الميزة (سجلها كـ "غير موجود").
- الدليل المطلوب: لقطات.

### BST-09 — حفظ الجلسة واستعادتها
- الخطوات: احفظ جلسة كاملة، أعد تحميل الصفحة، استعدها.
- المتوقع: استرجاع كامل للأفكار والوكلاء والحالة.
- الدليل المطلوب: لقطات قبل وبعد + Storage.

### BST-10 — تصدير نتائج الجلسة
- الخطوات: صدّر نتائج جلسة بكل صيغة متاحة.
- المتوقع: ملفات سليمة وقابلة للفتح.
- الدليل المطلوب: ملفات + لقطات.

### BST-11 — إدارة الوكلاء
- الخطوات: أضف وكيلًا، أزل وكيلًا، عدّل إعدادات وكيل إن أمكن.
- المتوقع: تطبيق التغييرات على الجلسة الحالية.
- الدليل المطلوب: لقطات + Network.

### BST-12 — البحث والفلترة داخل الأفكار
- الخطوات: ابحث عن كلمة أو فلتر.
- المتوقع: عرض النتائج المطابقة فقط.
- الدليل المطلوب: لقطات.

### BST-13 — جلسة طويلة بمدخلات كثيرة
- الخطوات: شغّل جلسة بـ 50 فكرة وأكثر من وكيل لمدة 10 دقائق.
- المتوقع: استقرار الواجهة وعدم تدهور الأداء.
- الدليل المطلوب: لقطات + Performance.

### BST-14 — انقطاع الشبكة أثناء انتظار الردود
- الخطوات: شغّل Offline في منتصف انتظار رد وكيل.
- المتوقع: رسالة واضحة وإمكانية استئناف بعد العودة.
- الدليل المطلوب: لقطة + Network.

### BST-15 — عدم كشف مفاتيح أو رسائل خطأ داخلية
- الخطوات: استثر خطأ وكيل وراقب Console.
- المتوقع: غياب أي مفاتيح أو stack trace أو معلومات خادم.
- الدليل المطلوب: لقطات + Console.

## 8) معايير قبول تنفيذ الاختبار

- لا يُقبل التقرير النهائي إلا إذا غطى كل الحالات في الأقسام 5 و 6 و 7.
- لا يُقبل تسجيل النجاح بدون دليل.
- لا يُقبل تسجيل الفشل بدون خطوات إعادة إنتاج.
- لا تُقبل الحالة المحجوبة إلا بعد توضيح ما جُرّب وسبب التعذر.
- لا يُقبل خلط هذا التطبيق مع أي تطبيق آخر داخل التقرير.
- لا يُقبل الحكم Production-Ready إلا إذا لم توجد إخفاقات حرجة أو مرتفعة غير محلولة.
- لا يُقبل أي فحص يكتفي بقراءة الكود أو نسخ نتائج تقرير قديم.
- يجب الالتزام بصيغة التقرير في القسم 4 حرفيًا.
