---

## 1) النطاق

- الاختبار محصور في تطبيق art-director فقط على الرابط أعلاه.
- ممنوع توسيع الاختبار إلى أي صفحة أخرى.
- ممنوع الخلط مع cinematography-studio أو styleIST حتى لو تشاركت بعض المفاهيم البصرية.
- أي انتقال إلى تسجيل الدخول أو رفع ملف أو تصدير يُسمح به فقط إذا انطلق من داخل التطبيق وكان مرتبطًا بوظيفة من وظائفه المباشرة.
- أي خلل خارج التطبيق لا يُسجل إلا إذا منع اختبار التطبيق نفسه بشكل مباشر.
- ممنوع تعديل كود التطبيق أو إصلاح أي عطل.

---

## 2) قواعد حاكمة غير قابلة للتفاوض

- لا تعتمد على قراءة الكود كدليل نجاح أو فشل، الدليل الوحيد المقبول هو ما يُلتقط فعليًا من المتصفح.
- لا تعتبر مجرد وجود زر أو عنصر دليل نجاح؛ يجب التفاعل معه والتحقق من نتيجة فعلية.
- لا تعتبر تحميل الصفحة وحده نجاحًا.
- لا تستخدم تقارير سابقة كدليل نجاح أو فشل.
- لا تُسقط حالة اختبار لأن الميزة غير واضحة؛ ابحث عن النظير الوظيفي.
- وثّق كل فشل بدليل مرئي أو سجل Console أو Network أو HAR.
- استخدم حالات: ناجح، فاشل، محجوب، غير موجود فقط.
- ممنوع استخدام "ناجح جزئيًا".
- ممنوع تشغيل Playwright أو Cypress أو أي إطار اختبار آلي.
- ممنوع استخدام Docker أو اقتراحه.

---

## 3) إعداد التنفيذ

- متصفح سطح مكتب حديث.
- دقة 1440 × 900، ثم اختبار بصري عند 768 × 1024.
- DevTools مفتوح: Console، Network، Storage، Cookies، Performance، Downloads.
- نافذة نظيفة أو Profile جديد لكل اختبار جلسات.
- استخدام Offline لاختبار انقطاع الشبكة.
- استخدام Axe DevTools أو Lighthouse للوصولية.
- التقاط لقطات شاشة قبل وبعد كل خطوة حاسمة.
- تسجيل HAR لكل فشل.

## 5) مصفوفة الاختبارات العامة

### GEN-01 — التحميل الأولي
- الخطوات: افتح الرابط في نافذة نظيفة.
- المتوقع: تحميل واجهة استوديو مدير الفن دون أخطاء حرجة.
- الدليل المطلوب: لقطة + Console + Network.

### GEN-02 — قابلية التفاعل الأساسية
- الخطوات: انقر على لوحة التحكم وعناصر التنقل.
- المتوقع: استجابة فورية لكل عنصر تفاعلي.
- الدليل المطلوب: لقطات + Console.

### GEN-03 — فحص Console
- الخطوات: ارصد Console من التحميل وحتى نهاية الجلسة.
- المتوقع: غياب أخطاء حمراء حرجة.
- الدليل المطلوب: نسخ كامل لمحتوى Console.

### GEN-04 — فحص Network
- الخطوات: راقب الطلبات الصادرة.
- المتوقع: غياب 4xx/5xx غير مبررة.
- الدليل المطلوب: HAR.

### GEN-05 — الوصولية
- الخطوات: شغّل Axe أو Lighthouse.
- المتوقع: غياب مشاكل Critical أو Serious.
- الدليل المطلوب: تقرير الأداة.

### GEN-06 — التنقل بلوحة المفاتيح
- الخطوات: استخدم Tab و Shift+Tab و Enter بين الأقسام.
- المتوقع: ترتيب منطقي للتركيز ووضوح حلقة التركيز.
- الدليل المطلوب: لقطات.

### GEN-07 — الاستجابة للشاشات الأصغر
- الخطوات: اضبط الدقة على 768 × 1024.
- المتوقع: لا انكسار للتخطيط ولا اختفاء وظائف رئيسية.
- الدليل المطلوب: لقطات.

### GEN-08 — الحفظ أو الاستمرارية
- الخطوات: أنشئ مشروعًا، أعد تحميل الصفحة.
- المتوقع: استرجاع المشروع أو رسالة واضحة بأنه مؤقت.
- الدليل المطلوب: لقطات + Storage.

### GEN-09 — LocalStorage و IndexedDB
- الخطوات: افحص ما يُكتب أثناء العمل.
- المتوقع: عدم تخزين بيانات حساسة بصيغة خام.
- الدليل المطلوب: لقطات Storage.

### GEN-10 — Cookies
- الخطوات: راقب الكوكيز قبل وبعد العمليات.
- المتوقع: HttpOnly و Secure و SameSite حيث يلزم.
- الدليل المطلوب: لقطات Cookies.

### GEN-11 — XSS في الحقول
- الخطوات: أدخل في حقول الأسماء والوصف:
  - `<script>alert(1)</script>`
  - `<img src=x onerror=alert(1)>`
  - `"><svg onload=alert(1)>`
- المتوقع: عرض النص حرفيًا أو رفضه.
- الدليل المطلوب: لقطة + Console.

### GEN-12 — إدخال طويل جدًا
- الخطوات: أدخل نصًا 50,000 حرف في حقل وصف موقع.
- المتوقع: عدم انهيار التطبيق.
- الدليل المطلوب: لقطة + Console.

### GEN-13 — انقطاع الشبكة
- الخطوات: شغّل Offline أثناء عملية حفظ.
- المتوقع: رسالة واضحة دون فقدان بيانات محلية.
- الدليل المطلوب: لقطة + Network.

### GEN-14 — التصدير إن كان مدعومًا
- الخطوات: صدّر التوثيق الفني.
- المتوقع: ملف صالح بامتداد وحجم معقول.
- الدليل المطلوب: ملف + لقطة.

### GEN-15 — الاستيراد أو رفع الملفات إن كان مدعومًا
- الخطوات: ارفع صورة صالحة، صورة بامتداد غير مدعوم، ملف فارغ، ملف ضخم.
- المتوقع: قبول الصالح ورفض البقية برسائل واضحة.
- الدليل المطلوب: لقطات + Network.

### GEN-16 — المصادقة
- الخطوات: ادخل بحساب صالح، اختبر تسجيل الخروج.
- المتوقع: انتقالات سليمة دون تسرب جلسة.
- الدليل المطلوب: لقطات + Storage + Cookies.

### GEN-17 — الروابط والمشاركة
- الخطوات: انسخ روابط مشاريع داخلية إن وجدت.
- المتوقع: عمل الروابط في نافذة جديدة.
- الدليل المطلوب: لقطات.

### GEN-18 — الأداء
- الخطوات: قِس زمن تحميل أدوات التصميم.
- المتوقع: عدم وجود حواجز أداء حرجة.
- الدليل المطلوب: تقرير Performance.

### GEN-19 — حالات الخطأ
- الخطوات: حاول حفظ مشروع فارغ، أو رفع ملف غير صالح.
- المتوقع: رسائل خطأ مفهومة دون stack.
- الدليل المطلوب: لقطات.

### GEN-20 — عدم تسرب البيانات بين الجلسات
- الخطوات: استخدم في جلسة A، اخرج، افتح Profile جديد.
- المتوقع: غياب بيانات الجلسة السابقة.
- الدليل المطلوب: لقطات.

## 6) مصفوفة الاختبارات الخاصة بـ art-director

### ART-01 — تحميل التطبيق
- الخطوات: افتح الرابط وانتظر تحميل لوحة التحكم الكاملة.
- المتوقع: ظهور dashboard و tools و inspiration دون أخطاء.
- الدليل المطلوب: لقطة + Console.

### ART-02 — التنقل بين الأقسام الرئيسية
- الخطوات: انتقل بين dashboard و tools و inspiration وأي قسم آخر متاح.
- المتوقع: انتقال سلس دون وميض غير مبرر.
- الدليل المطلوب: لقطات + قياس زمن.

### ART-03 — إنشاء موقع أو مشروع بصري جديد
- الخطوات: أنشئ موقعًا أو مشروعًا، أدخل البيانات الأساسية، احفظ.
- المتوقع: ظهور المشروع في القائمة مع تأكيد الحفظ.
- الدليل المطلوب: لقطات قبل وبعد + Network.

### ART-04 — تعديل عنصر تصميم
- الخطوات: عدّل عنصر تصميم داخل مشروع، احفظ.
- المتوقع: تطبيق التعديل وحفظه دون فقدان التغييرات.
- الدليل المطلوب: لقطات + Storage.

### ART-05 — استخدام أدوات تصميم الديكور
- الخطوات: شغّل أدوات تصميم الديكور وجرّب وظائفها الأساسية.
- المتوقع: استجابة الأدوات وظهور النتائج.
- الدليل المطلوب: لقطات + Console.

### ART-06 — إنشاء moodboard أو inspiration board إن وجد
- الخطوات: أنشئ board، أضف عناصر، احفظ.
- المتوقع: حفظ الـ board مع كل عناصره.
- الدليل المطلوب: لقطات + Storage.

### ART-07 — رفع أصل بصري إن كان مدعومًا
- الخطوات: ارفع صورة JPG، PNG، ثم ملف فيديو، ثم ملف غير مدعوم.
- المتوقع: قبول المدعوم ورفض البقية برسائل واضحة.
- الدليل المطلوب: لقطات + Network.

### ART-08 — حفظ المشروع ثم إعادة التحميل
- الخطوات: احفظ مشروعًا كاملًا، أعد تحميل الصفحة، افتح المشروع.
- المتوقع: استرجاع كل البيانات بدقة.
- الدليل المطلوب: لقطات قبل وبعد + Storage.

### ART-09 — تصدير التوثيق الفني
- الخطوات: صدّر التوثيق بكل الصيغ المتاحة.
- المتوقع: ملفات سليمة وقابلة للفتح.
- الدليل المطلوب: الملفات + لقطات.

### ART-10 — اختصارات لوحة المفاتيح
- الخطوات: جرّب كل اختصار موثق (Ctrl+S، Ctrl+Z، إلخ).
- المتوقع: تنفيذ كل اختصار بسلوكه المتوقع.
- الدليل المطلوب: لقطات قبل وبعد لكل اختصار.

### ART-11 — مشروع كبير بعدة مواقع وعناصر
- الخطوات: أنشئ مشروعًا بعشرة مواقع وعناصر تصميم متعددة.
- المتوقع: استقرار التطبيق وعدم تدهور الأداء.
- الدليل المطلوب: لقطات + Performance.

### ART-12 — المشاركة أو التعاون إن وجد
- الخطوات: شارك مشروعًا مع مستخدم آخر إن أمكن.
- المتوقع: عمل الميزة أو حالة "غير موجود" مع لقطة الواجهة.
- الدليل المطلوب: لقطات.

### ART-13 — رفض ملف غير مدعوم
- الخطوات: حاول رفع ملف بامتداد .exe أو .zip في حقل صورة.
- المتوقع: رفض واضح برسالة مفهومة.
- الدليل المطلوب: لقطة + Console.

### ART-14 — الأداء مع أصول بصرية كبيرة
- الخطوات: ارفع صورًا عالية الدقة (>10MB) متعددة.
- المتوقع: التطبيق لا ينهار، يعرض حدًا أو يعالج بكفاءة.
- الدليل المطلوب: لقطات + Performance.

## 8) معايير قبول تنفيذ الاختبار

- لا يُقبل التقرير النهائي إلا إذا غطى كل الحالات في الأقسام 5 و 6 و 7.
- لا يُقبل تسجيل النجاح بدون دليل.
- لا يُقبل تسجيل الفشل بدون خطوات إعادة إنتاج.
- لا تُقبل الحالة المحجوبة إلا بعد توضيح ما جُرّب وسبب التعذر.
- لا يُقبل خلط هذا التطبيق مع أي تطبيق آخر داخل التقرير.
- لا يُقبل الحكم Production-Ready إلا إذا لم توجد إخفاقات حرجة أو مرتفعة غير محلولة.
- لا يُقبل أي فحص يكتفي بقراءة الكود أو نسخ نتائج تقرير قديم.
- يجب الالتزام بصيغة التقرير في القسم 4 حرفيًا.
