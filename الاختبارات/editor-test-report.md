## 3) قائمة الأعطال المؤكدة بالأدلة

### **عطل 1 — حرج/قاطع: PDF Export معطل تماماً**

```
Error: Attempting to parse an unsupported color function "oklch"
```

ميزة التصدير لا تنتج أي ملف PDF. الإشعار يتكرر 6 مرات في DOM. هذا عطل قاطع لأن السيناريوهات تحتاج للتصدير لمشاركتها، ولأن أحد أهم استخدامات التطبيق معطّل.

**السبب الجذري المرشّح:** المكتبة المستخدمة في تحويل HTML إلى PDF (مثل html2canvas + jsPDF أو puppeteer-headless في عميل) لا تتعامل مع CSS Color Module Level 4 الحديث. الـ design system للموقع يستخدم `oklch()` بكثافة.

### **عطل 2 — حرج: schema تصنيف العناصر مكسور دلالياً**

البنية الناتجة من autosave HTML:

```html
<div data-type="basmala" class="screenplay-basmala">
  المشهد الأول داخلي – غرفة ضيقة – ليل
</div>
<div data-type="scene_header_top_line" class="screenplay-scene_header_top_line">
  <div data-type="scene_header_1" class="screenplay-scene_header_1">
    تدخل سلمى حاملةً مصباح تخييم
  </div>
  <div data-type="scene_header_2" class="screenplay-scene_header_2">
    سلمى أين أنت؟ أجبني.
  </div>
</div>
```

**التصنيف غير قياسي ومُربك**:

- `basmala` (البسملة - دعاء افتتاحي إسلامي) مُستخدم لعنوان المشهد (slugline). تسمية دينية لا تنتمي إلى مفردات السيناريو الصناعية.
- لا يوجد نوع `scene_heading` أو `slugline` أصلاً.
- لا يوجد نوع `action` لوصف الحركة.
- لا يوجد نوع `character` للشخصية المتكلمة.
- لا يوجد نوع `dialogue` للحوار.
- `scene_header_top_line` كحاوية تحوي `scene_header_1` (الحركة) و`scene_header_2` (الحوار) — تسمية مضللة لأن `top_line` يوحي بسطر علوي واحد لا حاوية.

**الأثر:** كل خوارزميات التحليل والإحصاء والتصدير ستعمل على schema غير قياسي، وأي صناعة محتوى ستتراكم على بنية لا يمكن نقلها إلى صيغ صناعية مثل Final Draft FDX أو Fountain.

### **عطل 3 — حرج: فقدان النص الطويل صامتاً**

محاولة إدخال 1500-5500 حرف عبر `document.execCommand('insertText')`:

| المحاولة | الإدخال | بعد لحظة | بعد 3-4 ثوانٍ |
|---|---|---|---|
| 1 | 5440 حرف | 5556 ظاهر | 113 (مفقود) |
| 2 | 1584 حرف | 1700 ظاهر | 113 (مفقود) |

النص يظهر، ثم يختفي تلقائياً، بدون إشعار، بدون رسالة خطأ. autosave يبقى عند 554 بايت. آلية revert صامتة. هذا فقدان بيانات نقطي.

### **عطل 4 — متوسط: زر "فتح ملف" فشل صامت**

```
fileInputs: []
modalsOpen: 0
```

الزر لا يفتح dialog ولا يعرض input، لا يطلق أي تفاعل ظاهر. المستخدم يفترض أنه يعمل لكنه معطل.

### **عطل 5 — حرج: زر "إعادة تصنيف المستند" يُفسد بدلاً من الإصلاح**

قبل النقر:

```
[basmala, scene_header_top_line, scene_header_1, scene_header_2]
عداد المشاهد: 1
```

بعد النقر (مع إشعار "فحص التكامل مكتمل - تم تشغيل 1 دالة بنجاح"):

```
[action, action, dialogue]
عداد المشاهد: 0
```

عنوان المشهد بقي مصنفاً خطأ (action بدل scene_heading)، عداد المشاهد انخفض، البنية المتداخلة دُمّرت. الزر يضرّ ولا ينفع.

### **عطل 6 — متوسط: التنسيق المضمَّن لا يُحفظ**

`<strong>` و`<em>` يظهران فوراً عند `Ctrl+B/I` لكن لا يبقيان في autosave HTML. بعد reload يختفيان كلياً. أيضاً فقدان بيانات نقطي.

### **عطل 7 — منخفض: Chrome MCP type يفقد المسافات في النص العربي**

ملاحظة منهجية مهمة لاختبارات لاحقة: استخدام `type` في الـ MCP لإدخال نص عربي يفقد المسافات بين الكلمات (المخرَج: "المشهدالأولداخلي–غرفةضيقة–ليل"). هذا قيد على أداة الاختبار، **ليس على المحرر**. الحل المعتمد: `document.execCommand('insertText')` عبر `javascript_tool`.

## 5) أهم المخاطر التشغيلية المُكتشفة

**1.** **PDF Export معطل = ميزة منتج كبرى مشلولة.** السيناريوهات لا يمكن تصديرها. هذا عطل قاطع لمنتج كاتب سيناريو.

**2.** **Schema تصنيف غير صناعي = ديون تقنية متراكمة.** أي محاولة تكامل مع أدوات صناعة السيناريو الخارجية ستفشل، وأي إحصاء أو تحليل سيُبنى على فئات خطأ.

**3.** **فقدان البيانات الصامت = ثقة المستخدم تنهار.** المستخدم يكتب فقرة طويلة، يراها، ثم تختفي بدون إنذار. هذه أسوأ تجربة محرر ممكنة.

**4.** **Bold/Italic لا يُحفظان = autosave منقوص.** أي عمل تنسيقي يضيع عند reload، رغم أن المحرر يدعم التنسيق.

**5.** **زر إعادة التصنيف يضرّ.** المستخدم الذي يحاول إصلاح وضعه يجعله أسوأ.

**6.** **FCP=7572ms = أداء سيء جداً.** أبطأ من العتبة المقبولة بـ3 أضعاف. كل التطبيقات الـ12 على المنصة تعاني من نفس مشكلة BUNDLE الكبير المسجلة في `all-apps-e2e-summary.md`.

## 6) توصيات الإصلاح المرتبة حسب الأولوية

### **حرجة قاطعة (تمنع الإنتاج)**

**T1.** **إصلاح PDF Export فوراً:**
- تشخيص: المكتبة المستخدمة (الأرجح `html2canvas` + `jsPDF` أو `dom-to-image`) لا تدعم `oklch()`.
- حل قصير: استبدال كل `oklch()` في الـ design tokens المستخدمة في صفحة `/editor` بقيم `rgb()` أو `hsl()` للـ light/dark themes. أو تحويل ألوان stylesheet التصدير قبل التحويل.
- حل بنيوي: الترقية إلى مكتبة تصدير حديثة مثل `puppeteer` (server-side) أو `html2pdf.js` v0.10+ التي بدأت تدعم oklch.

**T2.** **إعادة تصميم schema تصنيف العناصر بالكامل:**
- استبدال `basmala` بـ `scene_heading` أو `slugline`.
- استبدال `scene_header_top_line + scene_header_1 + scene_header_2` بـ `action`، `character`، `dialogue`، `parenthetical` كأنواع منفصلة قياسية.
- إضافة `transition` للانتقالات (CUT TO, FADE OUT).
- توفير migration script لتحويل المستندات الموجودة من schema القديم للجديد بدون فقدان.

**T3.** **إصلاح فقدان النص الطويل:**
- تحقيق آلية الـ revert الصامتة: ابحث في كود التطبيق عن أي transaction abort أو invariant check يُلغي المحتوى.
- أو رفع الحد الأقصى لطول العنصر الواحد (الأرجح أن `basmala` تفرض حداً أقصى لأن البسملة دلالياً قصيرة).
- إضافة رسالة واضحة لو فعلاً يجب رفض إدخال طويل.

**T4.** **إصلاح أو إخفاء زر "إعادة تصنيف المستند":**
- إذا الميزة لا تعمل صحيحاً، أخفها حتى يتم إصلاحها.
- إذا تعمل، فمنطق التصنيف الذي تشغّله يصنّف كل شيء كـ`action` و`dialogue` فقط — يحتاج إعادة كتابة كاملة لتمييز slugline من action من character من dialogue.

### **مرتفعة**

**T5.** **حفظ التنسيق المضمَّن في autosave:**
- الـ marks (bold, italic) من Tiptap لا تُحوَّل في الحالة الحالية إلى HTML المُسجَّل. تأكد أن `editor.getHTML()` يُستدعى لـ autosave بدل serialization مخصص يُسقط الـ marks.

**T6.** **ربط زر "فتح ملف" بـ `<input type="file">` فعلي:**
- إنشاء مدخل ملف مخفي.
- ربط onchange بـ parser للصيغ المدعومة (txt, md, fountain, fdx).

**T7.** **ضمان توافق outline ولون التركيز مع التصدير:**
- بعد إصلاح T1، تأكد أن `:focus-visible outline` لا يستخدم `oklab/oklch`.

### **متوسطة**

**T8.** **خفض FCP إلى أقل من 2500ms:**
- مراجعة Tiptap bundle، تأجيل تحميل الإضافات غير الأساسية.
- code splitting لـ `tiptap-pagination` فقد لا يُحتاج فوراً.

**T9.** **إضافة رسائل خطأ موحَّدة**:
- أي فشل صامت يجب أن يُترجَم إلى toast واضح بسبب الفشل.

**T10.** **مراجعة سلوك "تبديل التنسيق المباشر":**
- النقر على هذا الزر أعطى إشعار "تم تفعيل وضع التركيز الذكي" — هل هذا متوافق مع المفهوم المتوقع؟

## 7) ملاحظات تنفيذية إلزامية

**عدد محاولات الاختبار:** كل اختبار EDT-XX نُفِّذ ميدانياً مرة واحدة على الأقل، مع تكرار EDT-06 مرتين بأطوال مختلفة لتأكيد سلوك revert.

**الأدلة المتوفرة:**
- 9 لقطات شاشة مرقمة (ss_15395x9dc → ss_7262tuius).
- 12 استدعاء `javascript_tool` بنتائج JSON كاملة محفوظة في سجل الجلسة.
- محتوى autosave LocalStorage مأخوذ مباشرة من المتصفح.
- بنية HTML للعناصر مع `data-type` و`class` attributes موثَّقة حرفياً.

**ما لم يُختبر في هذه الجولة (للشفافية):**
- اختبار Console errors لأن أداة Chrome MCP لا تعرضها كـ stream متواصل.
- اختبار Lighthouse score (أداء، إمكانية وصول، PWA).
- اختبار Cookies tracking ومخاطر الـ tracking.
- اختبار Offline mode فعلياً (Service Worker).
- اختبار Multi-tab sync لو نفس المستند مفتوح في tabين.
- اختبار حدود قاعدة البيانات (مستندات بأحجام متطرفة).
- اختبار API endpoints الخاصة بالمحرر (`/api/editor/*` لو وُجدت) — اقتصرت الجولة على UI testing.

**حدود أداة الاختبار:**
- `Chrome MCP type` يفقد المسافات في النص العربي (تم تجاوزه باستخدام `javascript_tool` و`document.execCommand`).
- `Chrome MCP javascript_tool` يحجب أحياناً نتائج تحوي بيانات حساسة (cookies/query strings) مع رسالة `[BLOCKED: Cookie/query string data]` — لم يؤثر على نتائج هذا الاختبار.

**الحكم الإجرائي النهائي:**
- **`/editor` غير صالح للإنتاج** بسبب:
  - 3 أعطال حرجة (PDF export، schema تصنيف، فقدان نص).
  - 1 فشل ميزة كاملة (فتح ملف).
  - 1 عطل سلوكي (إعادة تصنيف يضرّ).
  - 2 فقدان بيانات (تنسيق مضمَّن، نص طويل).
- **الحكم القديم** ("معاقة بحاجز مصادقة") يُلغى بهذا الحكم الجديد المبني على اختبار ميداني فعلي للسطح بعد تجاوز الحاجز.
- **التوصية:** عقد جولة إصلاح حرجة على T1-T4 قبل أي شحن، مع جولة اختبار E2E ثانية بعد الإصلاحات للتحقق.

## 8) لقطة بنية العناصر للمرجعية

```html
<div class="tiptap ProseMirror tiptap-pages-mop70ln3-7i80pcs">
  <!-- pagination header (auto-generated, contains "1." page indicator) -->
  <div data-tiptap-pagination="true" id="pages" contenteditable="false">
    <div class="tiptap-page-break" data-page-number="0">...</div>
  </div>
  
  <!-- المحتوى الفعلي -->
  <div data-type="basmala" class="screenplay-basmala">
    المشهد الأول داخلي – غرفة ضيقة – ليل
  </div>
  
  <div data-type="scene_header_top_line" class="screenplay-scene_header_top_line">
    <div data-type="scene_header_1" class="screenplay-scene_header_1">
      تدخل سلمى حاملةً مصباح تخييم
    </div>
    <div data-type="scene_header_2" class="screenplay-scene_header_2">
      سلمى أين أنت؟ أجبني.
    </div>
  </div>
```
