# العقد الرسمي للوكلاء

## قاعدة منع إضعاف ملفات الفحص والتحقق

هذه القاعدة يجب أن تكون مقروءة ومثبتة القراءة قبل أي تحليل أو تعديل أو تنفيذ.

يُحظر حظرًا قاطعًا تعديل أي ملف فحص أو تحقق أو اختبار أو تدقيق أو بناء أو أمن أو صحة تشغيلية بطريقة تجعل الفحص أكثر تساهلًا، أو أقل قدرة على كشف الخلل، أو أضيق تغطية، أو أسهل مرورًا.

يشمل ذلك الاختبارات، والفحص الساكن، وفحص الأنواع، والبناء، والفحص الأمني، والتحقق التشغيلي، واختبارات الدخان، واختبارات التكامل، وأوامر التحقق النهائية، وخطوط التكامل المستمر، وقواعد المطابقة أو الأنماط أو الاستثناءات أو العتبات أو المخططات.

ويُعد إضعافًا محظورًا حذف فحص أو تعطيله أو تجاوزه، أو تحويل فشله إلى تحذير غير مانع، أو إضافة استثناءات تقلل التغطية، أو تضييق نطاقه، أو تخفيف الأنواع أو المخططات أو القيود أو العتبات، أو إزالة تحقق من مسار فشل أو تكامل أو أثر جانبي مؤثر، أو استبدال تحقق سلوكي مباشر بفحص ساكن أضعف.

إذا كان ملف الفحص نفسه يحتوي خطأً، فالواجب إصلاح الخطأ مع الحفاظ على نفس مستوى الصرامة أو زيادته. وأي بديل لفحص قائم يجب أن يثبت أنه مساوي له أو أشد منه من حيث التغطية، والصرامة، ومسارات النجاح والفشل، والتكامل، والآثار الجانبية.

أي تعديل يمس ملفات الفحص والتحقق يجب أن يخضع بعده لتشغيل فعلي مباشر للفحص المتأثر، بالإضافة إلى مسار التحقق المرجعي المناسب، ولا يجوز إغلاق الجولة بمجرد قراءة الملفات أو مقارنة النصوص.

أي brief مطلوب في صورة 3 إلى 7 حقائق تشغيلية يجب أن يتضمن حقيقة صريحة تثبت قراءة قاعدة الفحوصات الحاكمة.

هذا الملف هو العقد الأعلى المختصر لأي وكيل يعمل داخل هذا المستودع.

لا يوجد مصدر أعلى للحقيقة داخل المستودع غير هذا الملف مع العقد التشغيلي الكامل الموجود في:

```text
.repo-agent/OPERATING-CONTRACT.md
```

## قبل أي تحليل أو تعديل أو تنفيذ

شغّل أولًا:

```text
pnpm agent:bootstrap
```

إذا كانت الأداة لا تدعم hooks أصلًا وبدأت الجلسة من داخل واجهة IDE:

1. اقرأ هذا الملف.
2. اقرأ:

```text
output/session-state.md
```

3. اقرأ فقط ما يلزم من:

```text
output/code-map/*
output/mind-map/*
```

4. أخرج brief قصير يثبت القراءة في صورة 3 إلى 7 حقائق تشغيلية، ويتضمن حقيقة صريحة تثبت قراءة قاعدة الفحوصات الحاكمة.
5. ثم فقط ابدأ العمل.

## قواعد حاكمة
- ممنوع استخدام npm أو yarn داخل المسار الرسمي للوكلاء.
- ممنوع الاعتماد على ذاكرة الحوار أو واجهة الأداة كمصدر للحالة.
- ممنوع إنشاء أو الإبقاء على محاكاة محلية لأي مكوّن له مقابل رسمي داخل `Aceternity UI`.
- عند اكتشاف محاكاة محلية لمكوّن من `Aceternity UI` يجب استبدالها بالمكوّن الرسمي من السجل المعتمد وتوحيد الاستهلاك على مصدر رسمي واحد.
- لا يُسمح بوجود تنفيذين للمكوّن نفسه: نسخة محلية ونسخة `Aceternity UI` الرسمية معًا، ويُسمح فقط بملف عبور رفيع إذا احتاجت بنية المشروع ذلك.
- كل مكوّن بصري يظهر داخل صفحة المحرر يخضع لمحور التكوين البصري للمحرر نفسه، لا لمركز الشاشة الخام، سواء كان من `Aceternity UI` أو من أي مكتبة أخرى أو من تنفيذ محلي.
- السيمترية المرجعية في صفحة المحرر تُقاس بين ورقة التحرير وشريط `Dock` والحيز الحر قبل الشريط الجانبي، ويُمنع ترك أي عنصر عائم أو مثبت أو طبقة تفاعلية على منتصف الشاشة إذا كان محور الورقة مختلفًا.
- المصدر الوحيد للحالة التشغيلية الحالية هو:

```text
output/session-state.md
```

- السجل التنفيذي الوحيد للجولات هو:

```text
output/round-notes.md
```

- السياق الحي المولد يوجد في:

```text
.repo-agent/AGENT-CONTEXT.generated.md
```

- ملفات IDE وملفات الأدوات مرايا فقط وليست مصدر حقيقة مستقلًا.
- أي طبقة معرفة أو استرجاع أو embeddings أو indexing أو context assembly داخل المستودع تخضع لنفس هذا العقد الأعلى.
- لا توجد حقيقة مستقلة لطبقة RAG خارج الحقيقة التشغيلية العامة للمستودع.
