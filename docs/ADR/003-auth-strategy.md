# ADR-003: مصادقة مخصصة عبر JWT + Cookies + CSRF

## التاريخ
2026-03-30

## الحالة
مقبول

## السياق
الخلفية تحتاج حماية API تقليدية، Socket.IO، وعمليات كتابة حساسة.

## القرار
استخدام:

- JWT للوصول
- Cookies للجلسة
- CSRF double-submit لحماية عمليات الكتابة

## البدائل المدروسة

| البديل | المزايا | العيوب |
|---|---|---|
| مصادقة مخصصة | مرونة كاملة مع Express وSocket.IO | مسؤولية أمنية أعلى |
| NextAuth/Clerk | أسرع في الإطلاق | لا يظهر استخدامها في الكود الحالي |

## النتائج
الحل متماسك مع الخلفية الحالية، لكنه يتطلب انضباطاً عالياً في تهيئة البيئة ومراجعة المسارات التطويرية.
