# ADR-0008: تكامل MCP Server

## الحالة

مقبول

## السياق

Model Context Protocol (MCP) هو معيار مفتوح طوّرته Anthropic يُتيح لنماذج AI الوصول إلى أدوات وموارد خارجية بطريقة قياسية. "النسخة" منصة إنتاج سينمائي تحوي معرفة متخصصة (تحليل السكريبتات، بنية المشاهد، معلومات الشخصيات) يمكن تعريضها لنماذج AI عبر واجهة موحّدة.

الخيار هنا هو: هل نبني واجهة AI خاصة لكل أداة، أم نعتمد بروتوكولًا قياسيًا يجعل الأدوات قابلة للاستخدام من أي عميل AI متوافق؟

## القرار

اعتمدنا **`@modelcontextprotocol/sdk 1.22.0`** لبناء MCP Server مستقل (`apps/backend/src/mcp-server.ts`) يعمل كعملية منفصلة عبر `StreamableHTTPServerTransport` على منفذ `MCP_PORT`.

الدليل من الكود:
- `apps/backend/src/mcp-server.ts` ينشئ `McpServer` مع `StreamableHTTPServerTransport` على `/mcp` endpoint.
- كل طلب يُنشئ transport جديدًا (`sessionIdGenerator: undefined`) لمنع تضارب معرّفات الطلبات.
- `"dev:mcp": "tsx watch src/mcp-server.ts"` و`"mcp": "tsx src/mcp-server.ts"` سكريبتان مستقلتان تماميًا عن الخادوم الرئيسي.
- `apps/web/package.json` يتضمن `"@modelcontextprotocol/sdk": "^1.10.0"` و`"@ai-sdk/mcp": "^1.0.0"` للتكامل من جانب العميل.
- `apps/backend/package.json` يتضمن `"@ai-sdk/mcp": "^1.0.25"` للربط مع Vercel AI SDK.

## البدائل المدروسة

| البديل | المزايا | العيوب |
|---|---|---|
| **API خاص لكل أداة AI** | سيطرة كاملة، لا تبعية على بروتوكول خارجي | كل عميل AI جديد يحتاج تكاملًا مخصصًا، لا قياسية |
| **LangChain Tools فقط** | مدمج بالفعل في المشروع | مقيّد ببيئة Python/JS LangChain، لا يُعرّض الأدوات لعملاء خارجيين |
| **OpenAPI Spec** | معيار واسع الانتشار، دعم أدوات ضخم | لم يُصمَّم خصيصًا للـ AI tool use، لا streaming مدمج |
| **Function Calling مباشر** | مدمج في كل مزود AI رئيسي | كل مزود له تعريف مختلف، لا إمكانية إعادة الاستخدام |

## النتائج

### إيجابية
- **قياسية**: أي عميل AI متوافق مع MCP (Claude Desktop، Cursor، أدوات Vercel AI SDK) يستطيع استخدام أدوات "النسخة" مباشرةً.
- **عزل كامل**: MCP Server عملية مستقلة تمامًا، لا تؤثر في الخادوم الرئيسي إذا فشلت أو أُعيد تشغيلها.
- **`enableJsonResponse: true`**: يُتيح استجابات JSON بدلًا من streaming عند الحاجة، مما يُبسّط تكامل العملاء البسيطة.
- **Helmet مدمج**: MCP Server يُطبّق security headers عبر `helmet()` حتى في نسخته الأولية.
- **توسّع مستقبلي**: إضافة أدوات جديدة (تحليل شخصية، مقارنة مشاهد، اقتراح كاستينج) يتم بتسجيل tool جديدة دون تعديل الخادوم الرئيسي.

### سلبية
- **المرحلة الأولية**: الكود الحالي يتضمن أدوات تجريبية (`add` لجمع رقمين، `greeting` كمورد) ولم تُضف بعد أدوات سينمائية متخصصة.
- **منفذ إضافي**: `MCP_PORT` (افتراضيًا 3000) قد يتعارض مع الخادوم الرئيسي إذا لم يُهيَّأ بشكل صريح.
- **لا مصادقة بعد**: MCP Server الحالي لا يُطبّق أي مصادقة على endpoint `/mcp`، مما يُشكّل ثغرة في بيئات الإنتاج.
- **نسختان مختلفتان من SDK**: الخادوم يستخدم `^1.22.0` والويب `^1.10.0`؛ قد يُسبب عدم توافق في بعض الميزات.

- `/apps/backend/package.json` (`@modelcontextprotocol/sdk`, `@ai-sdk/mcp`)
- `/apps/web/package.json` (`@modelcontextprotocol/sdk`, `@ai-sdk/mcp`)
