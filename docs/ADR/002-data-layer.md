# ADR-002: طبقة بيانات هجينة بين JSON محلي وDrizzle في الخلفية

## التاريخ
2026-03-30

## الحالة
مقبول

## السياق
الويب يحتوي route handlers محلية تخزن بيانات بعض الأدوات في JSON، بينما الخلفية مصممة لتعمل مع PostgreSQL + Drizzle.

## القرار
الإبقاء على طبقة هجينة حالياً:

- JSON محلي لمسارات واجهات معينة
- PostgreSQL/Drizzle للخلفية طويلة العمر

## البدائل المدروسة

| البديل | المزايا | العيوب |
|---|---|---|
| طبقة هجينة | تسمح بتشغيل أجزاء من الويب دون backend كامل | تزيد التشتت وتخلق مصادر حقيقة متعددة |
| DB مركزية فقط | اتساق أعلى | تتطلب إكمال schema الخلفية وربط جميع المسارات بها |

## النتائج
المرونة أعلى، لكن التوثيق والتتبع أصعب، والبيانات قد لا تتطابق بين الويب والخلفية.
