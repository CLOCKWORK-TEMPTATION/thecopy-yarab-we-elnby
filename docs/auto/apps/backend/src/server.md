### توثيق الدالة `delay`

**الوصف:**
تقوم هذه الدالة بإيقاف تنفيذ الكود مؤقتاً لعدد محدد من الملي ثانية (milliseconds). تعمل كغلاف (Wrapper) لدالة `setTimeout` داخل كائن `Promise`، وسبب وجودها الأساسي هو توفير آلية إيقاف (Sleep/Delay) نظيفة وقابلة للقراءة متوافقة تماماً مع أسلوب `async/await` في العمليات غير المتزامنة، بدلاً من الاعتماد على دوال الاستدعاء (Callbacks).

**المعاملات (Parameters):**
* **`ms`** (`number`): مدة التأخير المطلوبة بالملي ثانية.

**القيمة المُعادة (Returns):**
* **`Promise<void>`**: كائن `Promise` يكتمل (Resolves) بدون إرجاع أي بيانات بمجرد انقضاء المدة الزمنية المحددة.

**الحالات الحدية والاستثناءات (Edge Cases & Exceptions):**
* **القيم السالبة أو الصفر:** إذا تم تمرير قيمة `0` أو قيمة سالبة للمعامل `ms`، سيكتمل الـ `Promise` فوراً (في الدورة التالية لـ Event Loop).
* **تجاوز الحد الأقصى للرقم الصحيح:** ترث هذه الدالة قيود `setTimeout` في محرك JavaScript، حيث أن أقصى قيمة مسموحة لمعامل الوقت هي `2,147,483,647` (حوالي 24.8 يوماً). إذا كانت قيمة `ms` أكبر من هذا الرقم، سيحدث طفحان (Overflow) وعادةً ما تكتمل الدالة فوراً كأن القيمة صفر.
* **القيم غير الرقمية:** إذا تم تمرير `NaN` أو `Infinity` (حال تجاهل TypeScript في وقت التشغيل)، فإن `setTimeout` يعاملها كـ `0`.

**مثال للاستخدام:**

```typescript
async function fetchWithRetry() {
  console.log("جاري المحاولة الأولى...");
  
  // انتظار لمدة ثانيتين (2000 ملي ثانية) قبل إعادة المحاولة
  await delay(2000);
  
  console.log("جاري المحاولة الثانية بعد انقضاء مدة التأخير...");
}
```
