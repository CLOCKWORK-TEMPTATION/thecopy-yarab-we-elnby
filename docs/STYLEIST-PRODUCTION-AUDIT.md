## 1. Architecture Overview

| Layer | Path | Technology |
|-------|------|-----------|
| Frontend App | `apps/web/src/app/(main)/styleIST/` | Next.js (React), Three.js, Framer Motion |
| AI API Route | `apps/web/src/app/api/gemini/route.ts` | Next.js API Route → Google Gemini |

**Key Finding:** The StyleIST frontend communicates with a **Next.js API Route** (`/api/gemini`) for AI operations. It has **ZERO** direct connection to the Express backend for data persistence.

### 3.1 CRITICAL: `generateStressTestVideo` — Dead Function
**File:** `services/geminiService.ts` lines 140-148

```typescript

  videoUrl: "",
  report: "اختبار الإجهاد غير متاح حاليًا - يتطلب تكوين API إضافي",

```
**Impact:** Stress test feature is non-functional.

### 3.2 CRITICAL: Dashboard Data File Missing
**File:** `components/Dashboard.tsx` line 25  
**Problem:** Imports from `../data/mockHistoricalData` — **file does not exist**.  
**Impact:** Dashboard component crashes at import.

### 3.3 HIGH: No Backend Persistence for StyleIST
**Problem:** The entire StyleIST app has NO persistence layer. All data is lost on page refresh:
- Designs generated by AI
- Wardrobe items (custom uploads)
- Scene-costume continuity data
- Safety reports
- Tech pack results

### 3.4 MEDIUM: `new-feature.tsx` — Simulated Processing
**File:** `new-feature.tsx` lines 66-72

## 7. Remaining Recommendations

1. **Fallback URLs** (`placehold.co`) in `geminiService.ts` are acceptable as fallbacks when AI generation fails — these should remain but be logged as warnings.
2. **Default wardrobe** in `wardrobe.ts` should eventually be loaded from the database per-project.
3. **Scene data** in `FittingRoom.tsx` should load from the backend project's scenes once connected to a real project context.
4. **Export PDF** button in FittingRoom is UI-only — needs backend PDF generation endpoint.
